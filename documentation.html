Getting Started

System Overview

Caddy Manager is a specialized web orchestration platform designed to manage Caddy Server instances with tactical precision. It provides a visual interface for dynamic configuration, traffic monitoring, and security management.

  • Frontend: Powered by React 18, Vite, and tactical Tailwind CSS.
  • Backend: Node.js Express server acting as a secure Caddy Admin Proxy.
  • Database: SQLite provides a robust state repository for all management assets.

Initialize Site

The "Initialize Site" workflow is the core mechanism for deploying new services. It is divided into two primary phases: General Settings and Security Rules.

Choose Site Blueprint

Select a pre-configured template (Blueprint) specialized for your application type:

The Bridge

Reverse Proxy

Standard forwarding for Node.js, Python, or Go apps. Optimized for high-performance traffic relay.

The Phantom

Single Page App

Specialized handling for React, Vue, or Angular SPA deployments with index fallback.

The Artisan

PHP / FastCGI

Seamless integration for PHP-based applications via FastCGI (e.g., Laravel, WordPress).

The Vault

Static Storage

High-speed static file serving from local directories with optional browsing.

The Architect

Custom Config

Full control over raw Caddy configuration for advanced routing and modules.

General Settings

  • Entry Hostname: The public-facing domain (e.g., cloud.example.com).
  • Upstream Target: The internal destination where your app is running (e.g., localhost:8080).
  • Edge Request Rules (Headers): Add or modify HTTP headers at the edge before traffic reaches your app.
  • Identity Logo: An optional URL for the site's logo to brand it within the Sites & Domains dashboard.

Security Rules

  • Access Control: List specific IP addresses to Whitelist or Block.
  • DNS Provider: Choose between Standard (HTTP) or DNS-01 providers for SSL.
  • Traffic Normalization: Redirect wizards for Host-only or WWW-only consistency.

Port Forwarding (Vector)

Deploy Layer 4 streams to tunnel non-HTTP traffic (SSH, Databases, Game Servers) through your orchestration node.

Choose Stream Blueprint

Select a blueprint optimized for your network vector:

The Pipeline

TCP Stream

High-performance TCP relay for SSH, Databases (MySQL/PG), and custom binaries.

The Signal

UDP Stream

Specialized UDP packet forwarding for game servers, real-time media, and DNS nodes.

The Multi-Vector

Dual Channel

Advanced dual-stack TCP/UDP proxying for complex tunneling and VPN protocols.

Vector Settings

  • Vector Handle: A human-readable name for monitoring (e.g., SSH Jumpbox).
  • Listen Port: The port on this server that will accept connections (e.g., 2222).
  • Protocol: Toggle between TCP, UDP, or both simultaneously.
  • Upstream Host/Port: The final destination for the traffic (e.g., 10.0.0.5:22).

Access Rules (Security)

Harden your network vectors with Layer 4 IP filtering:

  • Whitelist (Allowed) IPs: Only these addresses can connect. If set, all other traffic is dropped.
  • Blacklist (Blocked) IPs: Specific addresses to reject. Takes precedence over the default allow-all state.
  • Security Note: IP changes are atomic and take effect immediately upon "Save Vector".

Sites & Domains

The Dashboard

Real-time monitoring and lifecycle management for all initialized sites.

  • Node Identity: Visual status pulses and branding icons for every hostname.
  • Tactical Actions: Edit, Renew SSL, or Delete sites with atomic precision.

Security Nucleus (PKI)

Cryptographic Authorities

Manage internal Certificate Authorities (CAs) for encrypted intranet orchestration.

  • Sync Engine: Real-time synchronization with Caddy's pki module.
  • Trust Protocols: Automate the installation of root trust across host environments.

Developer Tools

Tactical Terminal

Fully integrated web-based SSH terminal for direct shell interaction with the target host.

Caddy Admin Proxy

All UI actions are proxied through a hardened Node.js backend to Caddy's Admin API on localhost:2019.
