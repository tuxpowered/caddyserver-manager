# Contributing to Caddy Manager

First off, thank you for considering contributing to Caddy Manager! It's people like you that make Caddy Manager such a great tool.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue using the Bug Report template. Include as much detail as possible, including steps to reproduce and environment details.

### Suggesting Enhancements
We welcome ideas for new features! Please use the Feature Request template to suggest enhancements.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure your code adheres to our aesthetic guidelines:
   - Use adaptive glassmorphism for UI elements.
   - Use theme-aware CSS tokens.
   - Implement smooth animations with Framer Motion.
3. Document all new backend logic.
4. Verify your changes locally using `npm run dev`.
5. Submit a Pull Request with a clear description of the changes.

## Style Guidelines
- **Frontend**: Tailwind CSS, Framer Motion, adaptive blurred backgrounds.
- **Backend**: Pragmatic Node.js/Express, SQLite for persistence.
- **Architecture**: Keep the layers (Frontend, Backend, Engine) separated and modular.

## Questions?
Feel free to open a discussion or an issue if you have any questions!
